02 — Design Violations

P0 Critical · P1 High · P2 Polish

Setiap violation dipetakan ke prinsip Rams + lokasi spesifik + fix terukur.

🔴 P0 · Rams #8: Thorough Down to Last Detail

Spacing Chaos — 14 Unique Padding Values per Page

CRITICAL

index_new.html (14 padding vals), pendekatan.html (14), artikel-detail.html (13 margin-bottom vals). Semua 7 halaman terpengaruh.

Tidak ada spacing system. 14 unique padding values = pixel pushing arbitrary, bukan sistem. Braun TP1 menggunakan 3 spacing values saja. "Thorough down to last detail" berarti sistem, bukan chaos.

/* 6-STEP SPACING SCALE — ganti semua arbitrary padding/margin */ --space-1: 0.5rem; /* 8px — micro gap */ --space-2: 1rem; /* 16px — tight */ --space-3: 1.5rem; /* 24px — default */ --space-4: 2.5rem; /* 40px — section padding */ --space-5: 4rem; /* 64px — major sections */ --space-6: 7rem; /* 112px — hero/footer padding */ /* Apply to all sections — no more arbitrary values like 1.8rem, 2.2rem, 3.5rem */
Braun TP1 (Dieter Rams, 1959) — circular record player dengan 3 jenis spacing saja. Apple iOS Human Interface Guidelines menggunakan 8-point grid system. Notion.so menggunakan 4-step spacing scale secara konsisten di seluruh produk.
🔴 P0 · Rams #10: As Little Design as Possible

Color Sprawl — 22–24 Unique Hex Colors per Page

CRITICAL

index_new.html: 24 unique hex colors. artikel.html: 23. filosofi.html + yang-anda-dapatkan: 22. Token system ada tapi halaman masih hardcode hex langsung.

Rams #10: "Less, but better." Braun ET66 kalkulator menggunakan 4 warna saja. 24 unique colors di satu halaman = tidak ada design discipline. Setiap iteration menambah warna baru instead of referencing tokens.

/* ACTIVE COLOR PALETTE — MAX 6 SEMANTIC COLORS */ --emerald: #0C3C2D; /* structural accent */ --gold-warm: #B8860B; /* decorative accent */ --champagne: #EBDAB0; /* dark-mode text */ --charcoal: #1A1A1A; /* dark bg */ --cream: #F9F7F3; /* light bg */ --coral-cta: #C0392B; /* WA CTA only — isolated */ /* ZERO inline hex literals. All references via var(--token) */ /* filosofi.html: remove color: #555, #444, #ccc — replace with var(--text-dark-m) */
Braun ET66 Calculator (1987) — 3 warna (putih, abu, merah satu tombol). Apple.com homepage 2025 menggunakan 2 accent colors maksimum per halaman. Linear.app menggunakan token system ketat — 0 hardcoded hex di component layer.
🔴 P0 · Rams #3: Aesthetic + Rams #6: Honest

13 Hardcoded Low-Contrast Colors + Fake Urgency di artikel.html

CRITICAL

filosofi.html (6×: #555, #444), index_new.html (6×: #666, #ccc, #aaa), pendekatan.html (1×: #444), yang-anda-dapatkan.html (1×: #666). artikel.html: fake urgency language detected.

Rams #3 aesthetic = visual harmony. #555 on cream = 3.6:1 (WCAG fail). #666 on #0d0d0d = 2.7:1 (FAIL). Rams #6 honest = no deceptive patterns. Fake urgency ("terbatas", "segera") manipulates user — violates Rams honest design.

/* Replace ALL hardcoded low-contrast colors: */ color: #555 → color: var(--text-light-m) /* #8B8275 = 4.5:1 on cream */ color: #666 → color: var(--text-dark-m) /* #888888 = 4.5:1 on dark */ color: #444 → color: var(--text-light-s) /* #4A4440 = 7.8:1 on cream */ color: #ccc → color: var(--text-dark-s) /* #A89968 = 6:1 on dark */ color: #aaa → color: var(--text-dark-m) /* #888888 = 4.5:1 on dark */ /* artikel.html — remove fake urgency text entirely */ /* Replace "Kesempatan terbatas" → remove or reframe as fact: "Slot tersedia untuk bulan ini" */
Dieter Rams pada Braun: "Seorang desainer yang tidak jujur tidaklah desainer." Stripe.com dan Linear.app tidak pernah menggunakan urgency language — trust dibangun dari kualitas produk, bukan pressure.
🔴 P0 · Rams #2: Useful

Zero Pricing Info — User Cannot Evaluate Value Before Contact

CRITICAL

Semua 7 halaman. Tidak ada "Rp", "IDR", "price", "harga", "mulai dari" di satupun halaman.

Rams #2 useful: "a product has to fulfill its purpose." Purpose consultation website = help user decide. Tanpa pricing signal, user tidak bisa decide → high drop-off sebelum WA contact. "Contact us for pricing" = friction yang tidak perlu.

/* yang-anda-dapatkan.html atau index_new.html — tambahkan pricing signal */ /* Tidak harus exact price. Cukup: */ "Mulai dari konsultasi 90 menit." "Investment: dari Rp X juta" "atau signal: "Custom pricing based on scope"" /* Minimum: satu anchor pricing signal agar user tidak datang ke WA tanpa ekspektasi */
Ideo.org, McKinsey.com, high-end consultancy — semua punya minimal one pricing anchor atau "starting from" signal. Removes friction. Honest = useful. Transparency builds trust lebih dari mystery pricing.
🟡 P1 · Rams #5: Unobtrusive

Gradient Overuse — 5 Gradients di artikel.html, Text-Shadow di 2 Pages

HIGH

artikel.html: 5 linear-gradient instances. index_new.html: 4 gradients + text-shadow. filosofi.html: 3 gradients + text-shadow.

Rams #5: design harus "neutral and restrained" — memungkinkan produk berbicara sendiri. Text-shadow pada dark hero tidak menambah legibility (kontras sudah 11:1 tanpa shadow). Gradients sebagai decoration = obtrusive.

/* REMOVE: semua text-shadow decorative (bukan accessibility) */ text-shadow: 0 0 80px rgba(200,169,110,.15); → /* DELETE */ /* REDUCE: max 1 gradient per halaman (hero overlay saja) */ /* artikel.html: dari 5 → 1 (hero overlay hanya) */ /* Replace gradient dividers → solid 1px border var(--border-dark) */
Braun ET44 kalkulator — 0 gradients, 0 shadows pada UI elements. Apple.com menggunakan max 1 gradient per section. Framer.com website: gradient hanya di hero, semua section lain solid color.
🟡 P1 · Rams #10: As Little as Possible

Inline Style Sprawl — filosofi.html (22 inline styles), index_new.html (16)

HIGH

filosofi.html: 22 inline style attributes. index_new.html: 16. yang-anda-dapatkan.html: 8. Total: 57 inline styles across 7 halaman.

Inline styles = design decisions yang ter-scatter, tidak reusable, tidak maintainable. Setiap update harus hunt semua halaman. Rams #10: "Less" means juga less complexity in implementation.

/* MIGRATION: semua inline style → CSS class */ style="color: var(--gold-warm); margin-bottom: 1.2rem;" → class="section-label" /* defined once in CSS */ style="font-size: 2.2rem; font-family: var(--ff-d);" → class="stat-num" /* Target: <5 inline styles per page — hanya untuk truly one-off positioning */
Tailwind CSS philosophy: utility classes vs inline styles. Linear.app codebase: 0 inline styles di production. Design system maturity = CSS classes yang predictable dan reusable.
🟡 P1 · Rams #9: Environmentally Friendly

Missing lazy-load di 3 Halaman — Potential CLS

HIGH

faq.html, pendekatan.html, yang-anda-dapatkan.html — tidak ada loading="lazy" pada img tags.

Rams #9: efficient, no waste. Images tanpa lazy-load = unnecessary data transfer untuk above-fold user. Cumulative Layout Shift risk.

/* TAMBAHKAN ke semua <img> below-the-fold: */ <img src="..." alt="..." loading="lazy" decoding="async" /> /* Hero image: tetap eager (above fold) */ <img src="hero.jpg" alt="..." loading="eager" />

🟡 P1 · Rams #7: Durable

Glassmorphism di 7/7 Halaman — Risiko Aging 2-3 Tahun

HIGH

Nav bar di semua 7 halaman menggunakan backdrop-filter: blur(14px). Glassmorphism trend peaked 2021-2023, mulai terasa dated.

/* ALTERNATIVE — solid dark nav (timeless) */ nav { background: rgba(13,13,13,0.98); /* solid dark, no blur effect */ /* remove: backdrop-filter: blur(14px) */ } /* Atau: keep blur tapi kurangi intensity */ backdrop-filter: blur(6px); /* dari blur(14px) → blur(6px) */
Braun radios — solid bezels, tidak ada transparency gimmick. Apple awalnya trend-setter glassmorphism di iOS 7 (2013) tapi kemudian moderasi di iOS 17. Solid dark nav = more durable.

03 — Font System v3

DM Sans (Display) + Outfit (Body)

Visby CF dan Neue Swiss tidak tersedia di Google Fonts — diganti dengan closest Google Fonts equivalents. DM Sans: geometric, authoritative (bold). Outfit: elegant, minimal (9 weights).

Display — DM Sans

Kejelasan Strategis

Bazi Wealth Path

Geometric sans-serif. Variable font (9–40pt optical size). Bold 700 untuk headlines — authoritative tanpa theatrical. Closest Google Fonts equivalent untuk Visby CF. Akan bertahan 10+ tahun.

Body — Outfit

Framework Analisis

Kerangka analisis berbasis pola untuk entrepreneur dan profesional yang ingin memahami situasi bisnis dan wealth mereka dengan lebih jernih. Bukan ramalan. Rekomendasi yang bisa dieksekusi.

Clean geometric, 9 weights (100–900). Warm humanist curves vs cold geometric. Lebih elegant dari Inter, lebih accessible dari Neue Swiss. Excellent readability di cream zones.

Pendekatan · Framework Analisis

Bukan ramalan. Kerangka analisis untuk kejelasan keputusan bisnis.

Ini soal pola yang tidak terlihat dari dalam. Sebagian besar konten BaZi berhenti di permukaan. Kami tidak.

Artikel · Long-Form Reading

Mengapa Pola BaZi Masih Relevan di 2025

Sistem BaZi bukan tentang mistik. Ia adalah framework psikologi keputusan berbasis waktu kelahiran yang telah digunakan selama lebih dari 1.200 tahun. Yang berubah adalah cara kita mengaplikasikannya.

07 — Design Rules

Rams-Compliant — Non-Negotiable

✓ Always — Rams Principles

  • DM Sans 700 untuk semua display/headline
  • Outfit 400/500 untuk body dan UI
  • Max 6 semantic color tokens — zero hardcoded hex
  • Max 6 spacing steps dari scale — zero arbitrary px
  • Ghost button border min 2px solid
  • Nav active = 2px bottom border + gold color only
  • WA CTA = fire red, isolated, never adjacent to gold
  • Cream bg hanya di artikel dan reading zones
  • Nav + footer tetap dark di semua mode
  • loading="lazy" pada semua below-fold images
  • Max 1 gradient per halaman (hero overlay saja)
  • Pricing anchor signal minimal di satu halaman

✗ Never — Hard Violations

  • Hardcoded #555, #666, #444, #ccc, #aaa langsung di CSS
  • Fake urgency language ("terbatas", "segera bertindak")
  • Gold + Red dalam satu komponen berdekatan
  • Underline pada nav link (kecuali active 2px border)
  • Ghost button border rgba opacity rendah (< 2px solid)
  • Text-shadow sebagai dekorasi (non-accessibility)
  • Lebih dari 6 active hex colors per halaman
  • Lebih dari 6 unique padding values (gunakan scale)
  • Inline styles lebih dari 5 per halaman
  • href="#" tanpa destination yang jelas
  • Gradient sebagai decorative divider (pakai border)
  • Emerald digunakan sebagai primary CTA (tetap merah)
